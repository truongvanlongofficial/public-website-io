1. Scope

This policy applies to the Session Tracker mobile application, its in-app session monitoring features, local on-device profile and journal features, advertising and consent flows, crash reporting, analytics, and push messaging features.

What this app is designed to do

Session Tracker is designed to help users launch and monitor gaming sessions, review device and network signals, store optional session notes and preferences, and view summaries such as session history, focus score, pinned games, and readiness checks.

What this policy does not cover

This policy does not replace the privacy notices and policies of third-party services such as Google AdMob, Firebase, or other Google services that may process data through their own SDKs.

3. Data the App Accesses, Collects, Stores, or Shares

The app uses a combination of on-device processing, local storage, and third-party SDKs. The exact data involved depends on which features a user chooses to use.

A. Data stored locally on the device by app features

  • Tracked or selected game names and package names.
  • Pinned games chosen by the user.
  • Per-game profile preferences, such as notes, target ping, Wi-Fi preference, charging reminder, and focus reminder settings.
  • Session journal entries, including session start and end time, baseline and closing ping, baseline and closing FPS, closing CPU temperature, and a calculated focus score.
  • Basic local app state, such as launch counters and feature state preferences.

As implemented in the current app build, these feature records are stored locally on the user's device using app storage such as SharedPreferences and are used to power in-app features.

B. Data the app may access from the device for functionality

  • Installed launcher app or game inventory so the user can view, select, pin, and launch games from within the app.
  • Network state information, such as whether the device is connected and whether Wi-Fi is active, to support readiness checks and network-related flows.
  • Notification permission status and device notification handling for app notifications and Firebase Cloud Messaging notifications.
  • Approximate device performance and session metrics, such as RAM availability, CPU temperature signals where available, and in-app ping/FPS values used for monitoring and summaries.

C. Data processed by third-party SDKs or services included in the app

  • Advertising data and device identifiers processed by Google Mobile Ads / AdMob and related Google ad services, which may include advertising identifiers, IP address, device information, app interaction data, and consent status, subject to Google's own policies and product configuration.
  • Consent and privacy preference signals processed through Google's User Messaging Platform (UMP).
  • Analytics and app usage event data processed by Firebase Analytics.
  • Crash diagnostics, technical device and app state information, and Firebase-generated identifiers processed by Firebase Crashlytics.
  • Push messaging data, such as Firebase Cloud Messaging registration tokens and delivery-related metadata, when notification features are enabled or used.

4. How the App Uses Data

  • To show the user's tracked games and allow game launch flows inside the app.
  • To save user-selected pinned games and per-game session preferences locally on the device.
  • To run session journal, focus score, and recent session recap features.
  • To evaluate readiness checks based on local device state and user preferences.
  • To show ads, manage ad consent, and support app monetization.
  • To measure product performance, app stability, feature usage, and crash diagnostics.
  • To send or receive app notifications where configured.
  • To protect app integrity, investigate errors, and improve the app.

5. VPN / Session Routing Disclosure

Certain session features may request Android VPN permission and create a local VPN interface on the device. This is used for app session routing, DNS selection, and related session features implemented in the app.

Based on the current code implementation, the developer does not describe this feature as a promise to reduce lag, guarantee lower ping, or otherwise guarantee network performance outcomes.

Based on the current code implementation, the app is not intentionally described as reading the contents of a user's personal messages, emails, or web content for the Session Tracker product features. However, because VPN-related features are sensitive, users should grant VPN permission only if they understand and want to use the relevant session functionality.

6. Installed Apps / Game Inventory Disclosure

The app may access the device's launcher app or installed game inventory so that users can view tracked titles, pin favorites, launch games, and build per-game session profiles. This data is sensitive under Google Play policy. As implemented in the current app build, this inventory is used for app functionality and local game management features.

The developer does not state that this inventory is sold. If third-party SDKs are present in the app, those SDKs must also comply with Google Play policy and applicable law.
